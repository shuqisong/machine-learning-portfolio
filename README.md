# Machine Learning Workspace

A personal workspace for data science & machine learning projects — built to
learn, practice, and grow a portfolio of resume-worthy work.

## Setup

```bash
# 1. Activate the virtual environment
source .venv/bin/activate

# 2. (First time only) install dependencies
pip install -r requirements.txt

# 3. Launch Jupyter to work in notebooks
jupyter lab
```

To leave the environment later, run `deactivate`.

## Structure

```
Machine_Learning/
├── .venv/              # Virtual environment (not committed)
├── requirements.txt    # Python dependencies
├── README.md           # You are here
└── projects/
    └── 01-movie-recommender/   # First project: a movie recommendation system
```

Each project is self-contained and follows a standard data-science layout:

```
project/
├── README.md           # What the project is, results, how to run it
├── data/
│   ├── raw/            # Original, immutable downloaded data (never edited)
│   └── processed/      # Cleaned/transformed data, generated by code
├── notebooks/          # Jupyter notebooks for exploration & analysis
├── src/                # Reusable Python code (functions, pipelines)
├── models/             # Saved trained models
└── reports/figures/    # Charts and visualizations for the writeup
```

## Why this layout?

Real data scientists keep **raw data immutable**, separate **exploration**
(notebooks) from **reusable code** (`src/`), and make every result
**reproducible from code**. Data and models are git-ignored because they're
large and regenerable — only the *code* that produces them is committed.
