# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A personal data-science / ML learning workspace organized as a collection of independent
**Kaggle-based projects**, each living under `projects/<NN-project-name>/`. The owner is a
beginner building a portfolio — favor explaining the *why* behind code/decisions, not just
producing working code.

## Commands

```bash
# Activate the environment (always do this first)
source .venv/bin/activate

# Install/update dependencies
pip install -r requirements.txt

# Launch Jupyter to work in notebooks
jupyter lab
```

There is no build, lint, or test suite — work happens in Jupyter notebooks plus small
reusable modules under each project's `src/`. To run a project's data-download script
directly (without activating the venv):

```bash
.venv/bin/python projects/<NN-project-name>/src/download_data.py
```

## Architecture

**Per-project layout** (see `projects/01-movie-recommender/` as the template):

```
project/
├── README.md           # roadmap, "how to run", results — written for a recruiter audience
├── data/
│   ├── raw/            # original downloaded data — immutable, never hand-edited, git-ignored
│   └── processed/      # cleaned/derived data, generated by code, git-ignored
├── notebooks/          # exploration & analysis (numbered: 01_exploration.ipynb, ...)
├── src/                # reusable Python modules (e.g. download_data.py)
├── models/             # saved trained models — git-ignored, regenerable from code
└── reports/figures/    # charts/visualizations for the project writeup
```

Key conventions to preserve when adding to or creating projects:

- **Raw data is immutable and git-ignored.** It's fetched via each project's
  `src/download_data.py` (uses the Kaggle API/CLI — requires `~/.kaggle/kaggle.json`,
  documented in the project README's "Get the data" section). Never commit dataset files,
  `kaggle.json`, or trained model binaries — `.gitignore` at the workspace root already
  blocks `data/raw/*`, `data/processed/*`, `models/*`, `*.csv`, `kaggle.json`, etc., with
  `.gitkeep` placeholders preserving the empty directories.
- **Notebooks are numbered and tell a learning story.** Each stage (EDA → baseline →
  improved models → evaluation) gets its own notebook with markdown cells that explain
  *what* is being done and *why*, written for someone learning the concepts — not just
  terse code. `01_exploration.ipynb` is the template for this style.
- **`src/` holds reusable code**, not one-off analysis — e.g. data-download scripts,
  shared feature-engineering or evaluation helpers that multiple notebooks import.
- **Each project's README** documents its roadmap (a stage-by-stage table), Kaggle data
  setup instructions, how to run it, and a "Results" section meant to read well to a
  recruiter — keep this updated as stages are completed.
- **Jupyter kernel**: the venv is registered as `ml-workspace` (display name
  "Python (ML Workspace)"); notebooks should select this kernel. If a fresh clone needs
  it re-registered: `.venv/bin/python -m ipykernel install --user --name ml-workspace --display-name "Python (ML Workspace)"`.
- New projects should follow the same `data/`, `notebooks/`, `src/`, `models/`,
  `reports/figures/` structure and the same staged-roadmap approach in their README.
